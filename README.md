# ركائز HRMS — نسخة GitHub Pages

هذه نسخة ثابتة تعمل مباشرة من `index.html` ولا تحتاج Node.js على الاستضافة.

يتضمن المشروع الآن موقعين:

- نظام الإدارة في جذر المشروع.
- بوابة الموظفين والبصمة داخل مجلد `employee-portal`، ويمكن رفع محتويات هذا المجلد إلى مستودع GitHub Pages مستقل.

## النشر على GitHub Pages

1. أنشئ مستودعاً جديداً في GitHub.
2. ارفع جميع الملفات الظاهرة معاً إلى جذر المستودع؛ لا توجد مجلدات داخل المشروع.
3. افتح Settings ثم Pages.
4. اختر Deploy from a branch، ثم `main` والمجلد `/root`.
5. احفظ الإعدادات وانتظر ظهور الرابط.

## مهم

- أضف رابط GitHub Pages إلى Firebase Authentication > Authorized domains.
- روابط n8n موجودة في `config.js`.
- يجب أن يعمل Docker وn8n وCloudflare Tunnel على جهازك لكي تصل رموز واتساب.
- لا ترفع كلمات مرور أو مفاتيح سرية إلى المستودع.

## موقع الموظفين

- فعّل Anonymous Authentication في Firebase قبل تشغيل بوابة الموظفين.
- أضف نطاق موقع الموظفين إلى Authorized domains.
- بوابة الموظفين تستخدم نفس Realtime Database وStorage، وتدخل بواسطة رمز البصمة المحفوظ للموظف.

## الترجمة بالذكاء الاصطناعي وPDF

إعدادات النموذج المحلي موجودة في `config.js` داخل `localAI`. الإعداد الحالي يستخدم Ollama على المنفذ `11434` والنموذج المثبّت `qwen3.5:9b` عبر واجهة OpenAI المتوافقة.

خدمة Ollama المحلية مضبوطة للسماح لموقع GitHub Pages الحالي عبر CORS. إذا استُخدم موقع أو نطاق جديد لاحقاً، أضفه إلى متغيّر `OLLAMA_ORIGINS` في خدمة `com.rakaez.ollama` المحلية. نشر الجدول لا يتوقف إذا تعذرت الترجمة، لكن زر PDF العربي/الإنجليزي يظهر بعد نجاح الترجمة فقط.
